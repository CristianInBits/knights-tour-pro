# Knight's Tour Pro - Benchmark Report (JMH)

## Overview

This document presents the performance benchmarks for **Knight's Tour Pro**, measured using **JMH (Java Microbenchmark Harness)**. The tests evaluate the throughput (operations per millisecond) of different solving algorithms across varying board sizes.

## Benchmark Configuration

* **Tool**: JMH 1.37
* **Benchmark Mode**: Throughput (ops/ms)
* **Warmup Iterations**: 3
* **Measurement Iterations**: 5
* **Forks**: 1
* **Time per Iteration**: Auto-calibrated

## Algorithms Tested

* **BacktrackingSolver**: Recursive backtracking approach.
* **WarnsdorffSolver**: Heuristic-based solver following Warnsdorff's Rule.

## Analysis

### BacktrackingSolver

* **5x5 Board**: Efficient due to the limited search space.
* **6x6 Board**: Exhibits exponential growth in computation, leading to a performance drop by a factor of \~167x.

### WarnsdorffSolver

* **5x5 Board**: Slightly slower than backtracking due to heuristic overhead.
* **6x6 Board**: Maintains a much smoother degradation in performance, only dropping by half.

### Backtracking vs Warnsdorff (Comparative Insight)

| Board Size | Better Algorithm | Difference Factor |
| ---------- | ---------------- | ----------------- |
| 5x5        | Backtracking     | 6.5x faster       |
| 6x6        | Warnsdorff       | 13x faster        |

* For **small boards**, raw backtracking can outperform heuristic methods due to minimal overhead.
* For **larger boards**, Warnsdorff's linear time complexity shows superior scalability.

## Conclusions

* **BacktrackingSolver** is suitable for small-scale problems but becomes impractical as the board size increases.
* **WarnsdorffSolver** provides stable and scalable performance, making it the preferred method for larger boards.
* The benchmarks validate the need for hybrid approaches (Warnsdorff + Backtracking) for balancing efficiency and scalability.
